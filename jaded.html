Jaded

(Social Networking Chat App)

Project Details

Tools: MongoDB, Express, React-Native, Node.js, Context API, socket.io, Expo

Description: I originally wanted to make a multiplayer game, but to do that, I knew communicating data between clients in real-time would be important, and http requests would not be sufficient for the task. So, instead, I decided to try making a chat app first to help give myself practice using websockets and real-time communication. I eventually used Expo to create the ios and android builds which I then submitted to the Google Play and iOS App Stores

Challenges/Obstacles: A major obstacle I faced while working on this app was scalability and performance. My initial implementations for a chatroom would fetch all the messages, and then re-render the entire list whenever a new message was added. After sending a few dozen messages, this eventually caused some noticeable lag!

To remedy this, I read through the React documentation and made use of the getItemLayout prop on the FlatList component (which was used to render all messages). By calculating the expected height of each message based on the content (text/images), I improved performance by only rendering whichever rows of the FlatList were visible on the user's screen at the time. On top of that, I also made use of React's "memo" HOC, to help ensure that when new data was received (ie. new message added), only the elements whose props changed would be re-rendered, as opposed to re-rendering all items on the screen.

Additonally, I had my node.js server fetch only the last 20 messages upon entering a room, and then fetch previous messages when the user scrolled to the top of the list.

What I Learned: This project, being larger and more complicated than some of my others, helped teach me some valuable lessons in organization and scalability. Looking back, I think my greatest mistake while working on this was a lack of thorough planning and design during the initial phase. Over time, I added more features and complexity to the application than I orginally planned, and as a result, it became more difficult managing all of the data and files. However, using the Context API to manage my state and breaking things down into modular components helped alleviate these issues.
